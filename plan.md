# 과제 수행 기획안 — [09] DA 공통 비즈니스 인사이트 리포트

> 작성: 2026-07-01
> 대상 과제: `problem.md` — 마케팅 성과 raw CSV → 인사이트 리포트 자동 생성

---

## 1. 문제정의

### 배경
저는 8주간(W1~W8) 5개 채널(네이버광고·메타광고·카카오광고·오가닉·이메일)에 마케팅을 집행한 이커머스 기업의 DA입니다. 마케팅팀·경영진은 "어느 채널이 효율적인가, 무엇이 변했는가"를 매주 알고 싶어 하지만, 현재는 이 리포트를 매번 수작업(추출→계산→시각화→요약)으로 만들어 반나절 이상이 소요되고, Excel 수식 오류로 ROI·변화율이 틀릴 위험이 있으며, 결과물이 "수치 나열"에 그쳐 실행 가능한 인사이트가 부족합니다.

### 핵심 질문 (이 리포트가 답해야 하는 것)
1. 전체적으로 마케팅이 돈값을 하고 있는가? — 총지출·총매출·전체 ROI·총전환수
2. 5개 채널 중 예산을 늘려야 할 곳과 줄여야 할 곳은 어디인가? — 채널별 ROI 순위
3. 지난 2주(W7→W8) 사이 무엇이 급격히 달라졌고, 왜 주목해야 하는가? — 이슈 3가지 + WoW 변화율

### 제가 정의하는 "문제"
> raw CSV(280행, 계산되지 않은 지표만 존재)를 **정확한 계산(Python)**과 **행동 가능한 해석(Claude)**으로 분리해, 매주 반복 가능한 인사이트 리포트로 전환하는 것.

데이터 자체에 함정이 있다는 점도 문제의 일부입니다 — 결측 3건, 완전중복 1쌍, 이메일 매출 이상치 1건, 오가닉 spend=0(0으로 나누기), 그리고 네이버광고 W7이 실제로는 6일치만 존재(analyze.py 실행으로 발견). 이걸 못 거르고 계산하면 리포트 전체가 왜곡됩니다.

### 성공 기준 (채점 기준 = 완료 정의)
- **Basic (60점 이상 필수)**: 핵심 수치 요약 + 채널별 ROI 순위표 + 해석 있는 이슈 3가지 + `insight_report.md`
- **Standard (+30)**: 새 CSV를 넣어도 동일 품질로 재현되는 파이프라인 + 직접 설계한 주간 리포트 구성
- **Challenge (+30)**: ROI·이슈 근거 기반 예산 재배분 기획안 1~2건 + 우선순위 판단 기준

### 제약
- 계산은 반드시 Python(합계·ROI·변화율), 해석만 Claude — 이 분리를 어기면 감점
- 시각화 이미지·대시보드·예측모델·실시간 API 연동 제외, 텍스트 리포트(.md)만

## 2. 접근 전략

`.claude/skills/`에 `/analyze → /insight → /generate → /review` 4개가 이미 패키징되어 있음. 이 순서를 그대로 따르되, Standard·Challenge에서는 "사람이 순서대로 실행하는 가이드"를 "재실행 가능한 산출물"로 한 단계 더 끌어올린다.

### 🟢 Basic

| 단계 | 상태 | 할 일 |
|------|------|------|
| /analyze | ✅ 완료 (`src/analyze.py`) | 결측 3건·완전중복 1쌍·이상치 1건·오가닉 spend=0·네이버 W7 6일 결측 위치 확정됨 |
| /insight | ⬜ | `insight.md` 기준대로 ROI 순위 + WoW(W7→W8) 계산, 이슈 3가지 최종 확정 |
| /generate | ⬜ | 계산 코드를 `src/calculate.py`로 정리해 실행 → 출력값을 `output/template.md` 형식에 그대로 삽입해 `output/insight_report.md` 작성 |
| /review | ⬜ | `review.md` 체크리스트로 자가 채점, Basic 80점 미달 시 /insight·/generate로 되돌아감 |

**이슈 3가지는 다음으로 확정** (skill이 제시한 후보 중 선택, 근거는 decisions.md에 기록):
1. 메타광고 W5 광고비 급등(~3.5배) → ROI 급락(306%→124%) — 예산 과집행 의심
2. 카카오광고 W4 광고비 급락(~41%↓) — 집행 누락/중단 가능성
3. 오가닉 무비용 고전환 — 광고비 0인데 전환·매출 최상위 → 콘텐츠/SEO 자산의 전략적 가치

> 데이터 품질(중복·결측·이상치)은 "비즈니스 이슈"가 아니라 "집계 전 정제 과정"으로 보고 1번 섹션(데이터 개요)에서 투명하게 처리 사실만 명시. 이슈 3가지는 정제 이후에도 남는 **비즈니스 임팩트가 있는 변화**로 한정 — 그래야 "단순 수치 나열"이 아닌 해석이 됨.

### 🟡 Standard

- **파이프라인 재현성**: 기존 skill 4개는 사람이 순서대로 읽고 실행하는 가이드일 뿐 진짜 자동화는 아님. `src/pipeline.py` 하나를 만들어 `python src/pipeline.py <csv경로>` 실행만으로 정제→집계→WoW→`insight_report.md` 저장까지 한 번에 수행하도록 통합. 하드코딩된 값(이메일 이상치 날짜, 채널명 등)은 함수 인자/상수로 분리해 새 CSV에도 로직이 깨지지 않게 함.
- **주간 리포트 설계**: `insight_report.md`에 "왜 이 섹션·지표를 넣었는가"를 별도 섹션으로 추가. 경영진용(총ROI 1줄 요약 + 채널 순위 + 이슈 3가지 + 액션 1~2개)과 마케팅팀용(주차별 CTR/CVR 추이 + 예산 조정 제안)을 구분해 근거 제시 — `context/industry-news.md` 5번 섹션(분석 리포트 작성 기준)을 근거로 인용.

### 🔴 Challenge

- **예산 재배분 기획안**: `output/template.md`의 `[Challenge]` 섹션 형식을 사용해 1~2건 작성.
  - 기획안 1 (안): 메타광고 예산 축소(W5 과집행분 회수) → ROI가 검증된 이메일·네이버광고로 재배분
  - 기획안 2 (선택): 오가닉(SEO/콘텐츠) 투자 신설 — 광고비 0으로 이미 고전환인 채널의 자산화
- **우선순위 판단 기준 설계**: 자체 스코어 공식을 정의해 기획안 간 우선순위를 매김.
  - `우선순위 점수 = (ROI 개선폭) × (관련 매출/광고비 규모) ÷ (실행 난이도, 1~3)`
  - 점수가 높을수록 P0(즉시 실행), 낮을수록 P1(검토 후 실행)로 표기하고 근거를 함께 기술
