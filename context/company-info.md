# DA 공통 — 마케팅 성과 분석 도메인 컨텍스트

> 이 과제는 특정 기업에 종속되지 않습니다. **데이터 분석가(DA)가 어느 회사에서나 반복하는 직무 공통 과제** — 마케팅 성과 리포팅 자동화 — 를 다룹니다. 아래는 그 직무가 마주치는 문제와 도메인/시장 배경입니다.

## 왜 이 과제가 중요한가 (직무 공통 문제)

마케팅 성과 분석은 광고·콘텐츠·채널 투자 대비 실제 비즈니스 결과를 수치로 검증하는 작업입니다. DA는 이 리포트를 매주·매월 반복 생산합니다.

- **어떤 채널이 ROI가 높은가?** → 예산 재배분의 근거
- **어떤 주차에 성과가 급변했는가?** → 이슈 탐지 및 원인 분석
- **지난주 대비 무엇이 달라졌는가?** → 의사결정 속도를 좌우하는 핵심 보고 항목

이 리포트는 마케팅팀 주간 회의, 예산 계획, 채널 전략 조정에 직접 쓰입니다. 그러나 현실의 DA는 매 주기 **데이터 추출 → 계산 → 시각화 → 요약**을 손으로 반복하느라 반나절 이상을 쓰고, Excel 수동 계산에서 ROI·변화율 오류가 나며, 결과물은 "수치 나열"에 그쳐 인사이트가 부족합니다. 이 과제의 목표는 그 반복을 **계산은 Python(정확), 해석은 Claude(풍부)** 로 자동화하는 것입니다.

## 핵심 지표 정의

### CTR (Click-Through Rate, 클릭률)
- 공식: `clicks / impressions × 100`
- 의미: 광고/콘텐츠를 본 사람 중 실제로 클릭한 비율
- ⚠️ raw CSV에는 CTR 컬럼이 없습니다 — impressions·clicks로 직접 계산하세요.

### CVR (Conversion Rate, 전환율)
- 공식: `conversions / clicks × 100`
- 의미: 클릭한 사람 중 실제로 구매·신청·가입한 비율
- 업종별 평균: e-commerce 1~3%, B2B 서비스 3~8%

### ROI (Return on Investment, 투자수익률)
- 공식: `(revenue - spend) / spend × 100` (= 광고비 대비 순수익 비율)
- 본 과제 정의(순위 비교용): `revenue / spend × 100` — 매출이 광고비의 몇 %인지
- 의미: 광고비 대비 매출/순수익 비율. **반드시 Python으로 계산** (수작업 시 합계 오류)
- ⚠️ `spend = 0`인 채널(오가닉)은 ROI가 정의되지 않습니다 — "측정 불가/별도 표기" 처리.

### ROAS (Return on Ad Spend, 광고수익률)
- 공식: `revenue / spend`
- 의미: 광고비 1원 당 발생 매출. 3.0이면 1원 투자 → 3원 매출
- ROI와의 차이: ROAS는 비용을 빼지 않으므로 수익성 자체와는 다를 수 있음

## 채널별 특성

| 채널 | 성격 | 강점 | 약점 | 성과 측정 핵심 |
|------|------|------|------|----------------|
| **네이버광고** (검색) | 구매 의도 높은 키워드 타겟, CPC 방식 | 높은 CVR, 정확한 타겟팅 | 클릭단가 높음, 노출량 제한 | CTR + CVR + 클릭당 비용 |
| **메타광고** (인스타/페북) | 이미지·영상 중심 소셜 노출 | 넓은 도달, 리타겟팅 | 클릭 이탈 높음, 소재 소진 빠름 | 도달 + CTR + 빈도 |
| **카카오광고** (비즈보드/친구) | 카카오톡 기반, 모바일 친화 | 광범위한 도달 | 구매 의도 낮음, CVR 낮음 | 도달 + CTR + 인지도 |
| **오가닉** (자연유입) | 검색·직접·추천 등 비유료 유입 | **광고비 0**, 높은 충성도 | 통제 어려움, 단기 확장 한계 | 유입량 + CVR (ROI는 측정 불가) |
| **이메일** (CRM) | 기존 고객 대상 CRM 채널 | 낮은 비용, 높은 CVR/ROAS | 오픈율 하락, 스팸 필터 | 오픈율 + CTR + 수신거부율 |

## 이슈 탐지 기준

### 급증·급락 기준
- **전주 대비 ±50% 이상 변화** → 이슈로 분류
  - 광고비 50% 급증 → 예산 초과 집행 가능성
  - 매출/전환 50% 급락 → 랜딩페이지 오류·경쟁 심화·소재 소진 가능성

### 이상치 기준
- **특정 일/주 매출이 전환수 대비 비현실적으로 급등** → 입력 오류 가능성, 검증 필요
- **광고비가 같은 채널 평소의 수 배 이상** → 예산 과집행, 원인 확인
- **특정 채널·일자 데이터 결측** → 집행 중단·수집 오류 가능성

### 처리 방법
- **완전 중복 행**: 집계 전 제거 (그대로 두면 매출·전환 합계가 부풀려짐)
- **이상치**: 제거 또는 별도 표기 — 무엇을 왜 제외했는지 리포트에 명시
- **결측치**: 0으로 채우지 말고 "데이터 없음"으로 명시 (groupby 합산 시 자동 제외)

## Python 계산 권장 이유 (수식 오류 방지)

1. **합계 오류 방지**: 280행 데이터의 채널별 합산을 손/머리로 처리하면 누락·중복이 발생합니다.
2. **변화율 예외 처리**: 전주 대비 % 변화율은 기준값이 0이거나 결측일 때 예외 처리가 필요합니다 (오가닉 ROI = 매출/0).
3. **재현 가능성**: Python 코드는 누구나 재실행해 검증할 수 있어 리포트 신뢰도가 올라갑니다.
4. **역할 분리**: 계산은 Python이, **인사이트 해석은 Claude가** 담당 — 이것이 이 과제의 핵심 설계입니다.

### 최소 Python 코드 예시
```python
import pandas as pd

df = pd.read_csv('data/marketing_performance.csv')

# 0) 중복 제거 + 숫자형 변환
df = df.drop_duplicates()
for c in ['impressions','clicks','spend','conversions','revenue']:
    df[c] = pd.to_numeric(df[c], errors='coerce')

# 1) 채널별 합계
g = df.groupby('channel').agg(
    spend=('spend','sum'), revenue=('revenue','sum'),
    conversions=('conversions','sum'),
    impressions=('impressions','sum'), clicks=('clicks','sum')
).reset_index()

# 2) 파생 지표 (spend=0 채널은 ROI 측정 불가)
g['ROI'] = (g['revenue'] / g['spend'] * 100).where(g['spend'] > 0)
g['ROAS'] = (g['revenue'] / g['spend']).where(g['spend'] > 0)
g['CTR'] = g['clicks'] / g['impressions'] * 100
g['CVR'] = g['conversions'] / g['clicks'] * 100

print(g.sort_values('ROI', ascending=False).to_string(index=False))
```

## 참고 용어 사전

| 용어 | 정의 |
|------|------|
| **impressions (노출수)** | 광고/콘텐츠가 화면에 표시된 횟수 |
| **clicks (클릭수)** | 노출된 광고를 사용자가 클릭한 횟수 |
| **spend (광고비)** | 해당 일·채널에 집행된 마케팅 비용(원). 오가닉은 0 |
| **conversions (전환수)** | 구매·신청·가입 등 목표 행동이 일어난 횟수 |
| **revenue (매출)** | 해당 일·채널에 귀속된 매출(원) |
| **CTR** | 클릭률 = clicks / impressions × 100 |
| **CVR** | 전환율 = conversions / clicks × 100 |
| **ROI** | 투자수익률 = (revenue - spend) / spend × 100 |
| **ROAS** | 광고수익률 = revenue / spend |
| **AOV** | 객단가 = revenue / conversions (전환 1건당 매출) |
| **오가닉(Organic)** | 광고비를 쓰지 않은 자연 유입(검색·직접·추천 등) |
| **전주 대비(WoW)** | Week-over-Week. 직전 주 대비 증감율(%) |
